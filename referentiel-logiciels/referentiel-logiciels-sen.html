Pourquoi classifier ?

La LGPD exige que les institutions publiques garantissent la sécurité des données de leurs usagers.

Pour le SEN et le CEJEF, cela concerne les données des élèves utilisées dans les logiciels pédagogiques.

Risque légal

Amendes RGPD jusqu'à 250'000 CHF

Cloud Act

Les données sur serveurs US sont accessibles par la justice américaine

Devoir de protection

Données d'élèves mineurs sous notre responsabilité

LGPD — Loi fédérale sur la protection des données (Suisse) RGPD — Règlement général sur la protection des données (UE) Cloud Act — Loi américaine autorisant l'accès aux données sur infrastructure US

3 niveaux de classification

VALIDÉ

Usage autorisé

Données d'élèves autorisées. Création de comptes OK.

RESTREINT

Usage avec précautions

Enseignants uniquement. Pas de données d'élèves.

INTERDIT

Usage interdit

Ni enseignants ni élèves. Non conforme.

Règle clé : Le niveau global est déterminé par le critère le plus faible.

Périmètre : Logiciels pédagogiques uniquement. Pas de données sensibles.

Proposition : plateforme unique

Même classification LGPD, même base de logiciels, avec des tags différenciés pour chaque service.

Avantages

  • ✓ Cohérence : un logiciel = une seule classification
  • ✓ Mutualisation du travail de reclassification
  • ✓ Un seul outil pour enseignants SEN et SFP
  • ✓ Maintenance simplifiée
