01 / Introduction

Pourquoi optimiser ses images ?

"Un bon croquis vaut mieux qu'un long discours"

Une mauvaise gestion des images impacte le référencement de votre site et la patience de vos visiteurs.

3 secondes — c'est déjà une éternité pour un internaute !

Erreurs fréquentes

✗

Mauvais format d'image

JPG pour un logo, PNG pour une photo...

✗

Images trop lourdes

Pas de compression = pages lentes

✗

Images trop grandes

Photos de 5000px non redimensionnées

✗

Noms incompréhensibles

DC00345.jpg, IMG_2847.png...

✗

Pas de texte alternatif

Mauvais pour l'accessibilité et le SEO

02 / Formats

Utiliser le bon format

JPG

JPEG

Idéal pour les photos

  • ✓ Compression efficace
  • ✓ 100% navigateurs
  • ✗ Pas de transparence
PNG

PNG

Idéal pour les logos

  • ✓ Transparence
  • ✓ Sans perte (lossless)
  • ✗ Fichiers lourds
SVG

SVG

Idéal pour les icônes

  • ✓ Vectoriel (zoom ∞)
  • ✓ Très léger
  • ✗ Pas pour photos
GIF

GIF

Idéal pour les animations

  • ✓ Animations simples
  • ✗ 256 couleurs max
  • ✗ Fichiers lourds
WebP

WebP

Moderne (Google)

  • ✓ 25-35% plus léger que JPG
  • ✓ Transparence + animation
  • ✓ 97% navigateurs
AVIF

AVIF

Nouvelle génération

  • ✓ 50% plus léger que JPG
  • ✓ Meilleure qualité
  • ✓ ~90% navigateurs

💡 Recommandation 2026

WebP reste le choix le plus sûr (97% de support). AVIF offre une meilleure compression mais avec un support légèrement inférieur (~90%).

📊 Comparaison taille (photo 1MB en JPG)

JPG: 1 MB WebP: ~700 KB AVIF: ~500 KB
03 / Dimensions

Redimensionner ses images

Si vos images s'affichent au maximum sur 750px de large, pourquoi envoyer des images de 5000px ?

Il est rare de voir des images de plus de 1800px de large sur un site web.

Trouver la taille max utile

  1. 1. Ouvrir la page sur un grand écran
  2. 2. Inspecter l'image (F12 → Elements)
  3. 3. Vérifier sa taille réelle d'affichage
  4. 4. Redimensionner à cette taille (ou légèrement plus pour le Retina)

✨ Règle d'or

Toujours redimensionner avant d'envoyer sur le serveur.

Comparaison de poids

Photo originale (5000px) 8.5 Mo
Redimensionnée (1800px) 1.2 Mo
Optimisée (1800px + compression) 180 Ko

Gain : 98% de réduction de taille !

04 / Compression

Compresser ses images

Trouver le bon compromis entre taille, poids et qualité.

📏 Résolution (DPI)

Pixels par pouce — D'origine à 300 DPI

Pour le web, 72 ou 96 DPI suffisent largement.

🎚️ Qualité de compression

100% = qualité maximum

Généralement 60-80% offre un bon compromis.

⚠️ Attention

La compression est irréversible. Gardez toujours vos originaux !

Niveaux de qualité

100% Maximum — souvent inutile
80% Haute qualité — bon compromis
60% Bonne qualité — fichiers légers
< 50% Dégradation visible — à éviter

Processus recommandé

1 Redimensionner à la taille utile
2 Passer la résolution à 72 DPI
3 Compresser à 60-80%
4 Convertir en WebP si possible
05 / Nommage

Utiliser des noms descriptifs

Les robots de recherche lisent les noms de fichiers pour comprendre le contenu des images.

Un nom descriptif avec des mots-clés séparés par des tirets améliore votre référencement.

✓ Bonnes pratiques

  • • Utiliser des mots-clés pertinents
  • • Séparer par des tirets (-) pas des underscores
  • • Tout en minuscules
  • • Éviter les caractères spéciaux et accents
  • • Garder des noms concis mais explicites

✗ Mauvais noms

06 / Accessibilité

Le texte alternatif

L'attribut alt est essentiel pour l'accessibilité et le référencement.

♿

Accessibilité

Les lecteurs d'écran utilisent le texte alt pour décrire les images aux personnes malvoyantes.

🔍

SEO

Google utilise le texte alt pour comprendre et indexer le contenu des images.

🖼️

Fallback

S'affiche si l'image ne charge pas correctement.

Syntaxe HTML

<img src="etang-gruere.jpg"
     alt="Vue de l'étang de la Gruère depuis la berge" />

📝 Conseils pour le texte alt

  • • Décrire le contenu ou la fonction de l'image
  • • Utiliser des mots-clés pertinents
  • • Rester concis (125 caractères max)
  • • Ne pas commencer par "Image de..."

Bonus : figure + figcaption

<figure>
  <img src="chat.jpg"
       alt="Chat roux dormant" />
  <figcaption>
    Mon chat Pixel fait sa sieste
  </figcaption>
</figure>

Les légendes permettent d'ajouter du contexte visible pour tous.

07 / WordPress

Médias dans WordPress

WordPress génère automatiquement plusieurs tailles pour chaque image téléversée.

Champs importants

T

Titre

Pour la recherche interne WordPress. Utilisez des mots-clés !

A

Texte alternatif

Le plus important ! Décrit l'image pour l'accessibilité et le SEO.

L

Légende

Optionnel — s'affiche sous l'image si configuré.

D

Description

Optionnel — pour les pages d'attachement.

Tailles générées par WP

Miniature 150 × 150 px
Moyenne 300 × 300 px
Grande 1024 × 1024 px
Originale Taille d'upload

Configurable dans Réglages → Médias

⚠️ Important

WordPress utilise le titre comme texte alternatif par défaut si le champ alt est vide. Ce n'est pas une bonne pratique — remplissez toujours le texte alternatif !
